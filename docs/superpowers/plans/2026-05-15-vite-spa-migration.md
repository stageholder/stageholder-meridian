# Meridian Vite SPA Migration Implementation Plan

> **For agentic workers:** Per the user's saved preferences this plan contains NO TDD steps and NO `git commit` steps — the user runs git themselves. Bun (not node/npm/yarn/pnpm) is mandatory for every command.

**Goal:** Replace `apps/pwa` (Next.js 16 App Router + BFF + iron-session + MongoDB session store) with a pure Vite + React 19 SPA that talks to the Meridian API directly using `@stageholder/sdk/spa` bearer tokens. Unify `apps/desktop` (Tauri) onto the same source tree.

**Architecture:**

- Vite 7 + React 19.2 + TanStack Router v1 (file-based) + TanStack Query v5 (already used)
- `@stageholder/sdk` bumped from `1.0.0-alpha.42` → `1.0.0-alpha.45+` and consumed via `/spa` (`StageholderSpaProvider` + `createAuthenticatedFetch`)
- Tailwind v4 via `@tailwindcss/vite`
- All BFF routes (`/api/v1/[...path]`, `/api/me`, `/api/me/onboarding/complete`, `/api/account/[...path]`, `/api/billing/[...path]`, `/api/debug/token`, `/(auth)/auth/[...stageholder]`) deleted; SPA calls Meridian API + Hub directly with `Authorization: Bearer` injected by the SDK
- iron-session + MongoDB session store removed from PWA (MongoDB stays — used by Meridian API for app data)
- `apps/desktop` retires its `wait-on http://localhost:4001` boot dependency and mounts the Vite source tree directly via path alias (same pattern as `stageholder-atlas/apps/desktop`)
- PWA service worker switches from a hand-rolled `public/sw.js` registration to `vite-plugin-pwa` for auto-generated precache + manifest injection

**Tech Stack:** Vite 7, React 19.2, TanStack Router 1.85+, TanStack Query 5.62+ (existing), `@stageholder/sdk` 1.0.0-alpha.45+, Tailwind 4, `@tailwindcss/vite` 4, `vite-plugin-pwa`, TypeScript 5.9, Bun 1.3.

## Why this is the right move (one-time read)

The BFF was scaffolded before SDK SPA mode shipped. Today it is six pass-through route handlers (`/api/v1/[...path]`, `/api/account/[...path]`, `/api/billing/[...path]`, `/api/me`, `/api/me/onboarding/complete`, `/api/debug/token`) plus the SDK catch-all at `(auth)/auth/[...stageholder]`. Every one of them just forwards a request with `Authorization: Bearer` and zero meridian-specific logic — work the SDK's SPA mode does in-browser.

The MongoDB session store and `MeridianCustom` `enrichSession` hook exist because the BFF needs to cache `{ personalOrgId, hasCompletedOnboarding }` across requests. In SPA mode we cache the same payload in TanStack Query (`staleTime: 60_000`) — same outcome, no Mongo round-trip, no server hop. The Meridian API already exposes `GET /api/v1/me` returning exactly that shape (`apps/api/src/modules/me/me.controller.ts:44`).

The desktop app pays a real price for the current shape: `apps/desktop/package.json` boots with `wait-on http://localhost:4001 tcp:localhost:4000 && tauri dev` — two parallel dev servers, ad-hoc IPC, and `lib/oidc-tauri.ts` hand-rolling PKCE because the Next.js auth catch-all doesn't work inside Tauri. Post-migration, Tauri mounts the Vite source tree directly (same as atlas), shares one PKCE codepath via SDK SPA with a `TauriStorageAdapter`, and the `wait-on` step goes away.

We drop: `next`, `iron-session`, `mongodb` (PWA only — API keeps it), the `app/api/*` directory, `proxy.ts`, `next.config.ts`, `lib/stageholder.ts`, `lib/public-origin.ts`, and the entire `lib/oidc-tauri.ts` hand-rolled PKCE (~300 LOC). The Meridian API is untouched except for CORS.

import { ConfigError } from "@stageholder/sdk/spa";

function readEnv(key: string): string {
  const value = import.meta.env[key];
  if (typeof value !== "string" || value.length === 0) {
    throw new ConfigError(
      `Missing required env var ${key}. Set it in apps/pwa/.env.local.`,
    );
  }
  return value;
}

const issuerUrl = readEnv("VITE_IDENTITY_ISSUER_URL");

## Phase 2: API client replacement

### Task 2.1: ServiceWrapper

**Files:**

- Create: `apps/pwa/src/utils/service-wrapper.ts`

Thin axios-compatible wrapper around `createAuthenticatedFetch` from `@stageholder/sdk/spa`. Returns the `{ data, status, headers }` envelope so existing API modules in `lib/api/*.ts` don't have to change.

```ts
import {
  createAuthenticatedFetch,
  type ProviderBinding,
} from "@stageholder/sdk/spa";
import {
  MERIDIAN_PAYWALL_EVENT,
  type MeridianPaywallDetail,
} from "@/components/paywall-listener";

interface RequestConfig {
  params?: Record<string, unknown>;
  signal?: AbortSignal;
  data?: unknown;
}

interface Envelope<T> {
  data: T;
  status: number;
  headers: Headers;
}

export class ServiceError<T = unknown> extends Error {
  constructor(
    message: string,
    public response: { status: number; data: T; headers: Headers },
  ) {
    super(message);
    this.name = "ServiceError";
  }
}

export class ServiceWrapper {
  private fetcher: ReturnType<typeof createAuthenticatedFetch>;

  constructor(
    private baseUrl: string,
    binding: ProviderBinding,
  ) {
    this.fetcher = createAuthenticatedFetch({ binding });
  }

  async get<T>(path: string, config: RequestConfig = {}): Promise<Envelope<T>> {
    return this.request<T>("GET", path, undefined, config);
  }
  async post<T = unknown>(
    path: string,
    data?: unknown,
    config: RequestConfig = {},
  ): Promise<Envelope<T>> {
    return this.request<T>("POST", path, data, config);
  }
  async put<T = unknown>(
    path: string,
    data?: unknown,
    config: RequestConfig = {},
  ): Promise<Envelope<T>> {
    return this.request<T>("PUT", path, data, config);
  }
  async patch<T = unknown>(
    path: string,
    data?: unknown,
    config: RequestConfig = {},
  ): Promise<Envelope<T>> {
    return this.request<T>("PATCH", path, data, config);
  }
  async delete<T = unknown>(
    path: string,
    config: RequestConfig = {},
  ): Promise<Envelope<T>> {
    return this.request<T>("DELETE", path, config.data, config);
  }

  private async request<T>(
    method: string,
    path: string,
    body: unknown,
    config: RequestConfig,
  ): Promise<Envelope<T>> {
    const url = new URL(
      this.baseUrl.replace(/\/$/, "") +
        (path.startsWith("/") ? path : `/${path}`),
    );
    if (config.params) {
      for (const [k, v] of Object.entries(config.params)) {
        if (v !== undefined && v !== null) url.searchParams.set(k, String(v));
      }
    }
    const init: RequestInit = {
      method,
      signal: config.signal,
      headers: { "Content-Type": "application/json" },
    };
    if (body !== undefined && method !== "GET" && method !== "HEAD") {
      init.body = JSON.stringify(body);
    }

    const res = await this.fetcher(url.toString(), init);
    const contentType = res.headers.get("content-type") ?? "";
    const data = contentType.includes("application/json")
      ? await res.json()
      : await res.text();

    if (res.status === 402 && typeof window !== "undefined") {
      const detail = (data as Partial<MeridianPaywallDetail>) ?? {};
      window.dispatchEvent(
        new CustomEvent(MERIDIAN_PAYWALL_EVENT, {
          detail: {
            code: detail.code ?? "feature_limit",
            feature: detail.feature ?? "unknown",
            featureLabel: detail.featureLabel,
            limit: detail.limit ?? 0,
            current: detail.current ?? 0,
            suggestedPlan: detail.suggestedPlan,
            suggestedPlanName: detail.suggestedPlanName,
          },
        }),
      );
    }

    if (!res.ok) {
      throw new ServiceError(`${method} ${path} failed: ${res.status}`, {
        status: res.status,
        data: data as T,
        headers: res.headers,
      });
    }

    return { data: data as T, status: res.status, headers: res.headers };
  }
}
```

const STORE_PATH = "auth.dat";

export class TauriStorageAdapter implements StorageAdapter {
  async getItem(key: string): Promise<string | null> {
    const s = await store();
    const value = await s.get<string>(`${KEY}:${key}`);
    return value ?? null;
  }
  async setItem(key: string, value: string): Promise<void> {
    const s = await store();
    await s.set(`${KEY}:${key}`, value);
    await s.save();
  }
  async removeItem(key: string): Promise<void> {
    const s = await store();
    await s.delete(`${KEY}:${key}`);
    await s.save();
  }
}

### Task 11.2: nginx.conf

**Files:**

- Create: `apps/pwa/nginx.conf`

```nginx
server {
  listen 4001;
  root /usr/share/nginx/html;
  index index.html;

  # SPA fallback — all non-asset paths return index.html.
  location / {
    try_files $uri $uri/ /index.html;
  }

  # Hashed assets get 1-year immutable cache.
  location ~* \.(js|css|woff2?|png|jpg|jpeg|svg|webp)$ {
    expires 1y;
    add_header Cache-Control "public, immutable";
  }

  # index.html and the service worker must never be cached.
  location = /index.html {
    add_header Cache-Control "no-store, max-age=0";
  }
  location = /sw.js {
    add_header Cache-Control "no-store, max-age=0";
  }
}
```

The PWA service now exposes nginx (port 4001) instead of `next start`. If the cluster runs both behind a gateway, the gateway config doesn't change — the upstream container just serves different bytes.
